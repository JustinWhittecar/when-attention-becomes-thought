# Reading notes

One file per chapter. One entry per paper, using the structure in `TEMPLATE.md`.

These notes are the raw material for the book. They are not polished. They are public (or private, your call) so that readers can see the work and other researchers can catch errors.

Workflow:

1. Reading night: pick the next paper in the chapter's reading list. Fill in all seven fields of the template. Do not skip "Teaching angle."
2. Synthesis night: re-read the chapter's notes file top to bottom. Fill in the Synthesis block at the bottom.
3. Writing night: open a new chapter file in `src/`. Write from the notes, not the papers.

Every paper in `notes/` should have a matching citation in `src/references.md` by the time the chapter is drafted.
